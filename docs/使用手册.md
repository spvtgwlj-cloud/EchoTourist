## 1. 系统概要

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Next.js 15 (React) | SSR + ISR，多语言路由 |
| 后端 | FastAPI (Python 3.12) | 异步 API，Route → Service → CRUD 三层 |
| 数据库 | PostgreSQL 16 | 主存储 |
| 缓存 | Redis 7 | API 响应缓存（60-300s TTL） |
| 搜索 | Elasticsearch 8.17 | 全文搜索 |
| 消息队列 | Celery + Redis | 异步任务（邮件、搜索索引） |
| 邮件调试 | MailHog（profile 模式） | 开发环境邮件捕获（`docker compose --profile mail up`） |
| 容器化 | Docker Compose | 10 个服务容器（含 nginx + mailhog） |

### 服务架构

```
nginx (80) ← 统一入口，反向代理
  ├─ frontend (Next.js 3000) — SSR 渲染，通过 next.config.ts rewrites 代理 /api 到 backend
  └─ backend (FastAPI 8000)  — REST API
       ├─ postgres (5432)
       ├─ redis (6379)
       ├─ elasticsearch (9200)
       ├─ celery_worker — 异步任务
       └─ celery_beat  — 定时任务
```

> ⚠️ **日常使用请直接访问 `http://localhost`**（走 nginx 代理）。`http://localhost:3000` 主要用于开发调试。

### 访问地址

| 服务 | 地址 | 说明 |
|------|------|------|
| 🌐 **网站首页** | **`http://localhost`** | **Nginx 统一入口（推荐）** |
| 🖥️ 前端（直连） | `http://localhost:3000` | Next.js SSR 开发模式 |
| ⚙️ 后端 API | `http://localhost:8000` | FastAPI，文档 `/docs` |
| 🔐 管理后台 | `http://localhost/en/admin` | 登录后自动跳转仪表盘 |
| 📧 MailHog（邮件调试） | `http://localhost:8025` | 开发环境捕获所有外发邮件（需 `--profile mail`） |
| 🇨🇳 中文版 | `http://localhost/zh` | 中文界面 |
| 🇬🇧 英文版 | `http://localhost/en` | 英文界面 |

## 3. 系统管理

### 启动服务

```bash
# 启动全部 10 个容器（含 nginx 反向代理）
cd /Users/wulianjun/.claude/Echo-Website
docker compose up -d

# 如需启用 MailHog（邮件调试）：
docker compose --profile mail up -d

# 查看服务状态（应有 10 个 Running，含 mailhog 为 11 个）
docker compose ps

# 首次启动后等待 10-20 秒让前端编译完成
# 然后打开 http://localhost 即可访问

# 查看日志
docker compose logs -f backend   # 后端日志
docker compose logs -f frontend  # 前端日志
docker compose logs -f nginx     # Nginx 日志
docker compose logs -f nginx | grep -E "error|warn"  # 只查看 Nginx 错误
docker compose logs -f mailhog   # 邮件捕获日志（profile 模式）
```

### 单独重启某个服务

```bash
# 修改代码后重启后端（开发模式会热重载，通常无需重启）
docker compose restart backend

# 修改 nginx 配置后重载（无需重启容器）
docker compose exec nginx nginx -s reload

# 重启前端
docker compose restart frontend
```

### 初始化种子数据

种子数据包含 12 个旅游产品、3 个目的地、45 个景点、4 个用户、12 条评论、55 个团期。

```bash
docker compose exec backend python /app/scripts/seed_data.py
```

脚本可重复执行（自动清空重建），不会产生重复数据。

### 运行测试

```bash

docker compose exec backend pytest tests/ -v

# 运行回归安全门禁（CI 质量关卡 — 逆向/边界/回归/安全测试）

# 运行指定测试文件

docker compose exec backend pytest tests/test_regression.py -v      # 跨模块回归
docker compose exec backend pytest tests/test_security.py -v        # 安全渗透

# 运行特定测试类
docker compose exec backend pytest tests/test_crud/test_review_crud.py -v

# 前端 E2E 测试（需在本地安装 Playwright）
cd /Users/wulianjun/.claude/Echo-Website
npx playwright test --config=tests/e2e/playwright.config.ts

# 运行 E2E 回归测试
npx playwright test tests/e2e/regression-*.spec.ts --reporter=list

# Python 全业务流程自动化（16 步有头模式）
cd /Users/wulianjun/.claude/Echo-Website

```

### 数据库迁移


## 4. 产品管理操作指南

### 访问管理后台

1. 打开 `http://localhost` → 右上角 **Sign In**
2. 使用管理员账号登录：`admin@echotours.com` / `Admin123!`
3. 登录后右上角会出现用户头像菜单 → 点击展开菜单
4. 管理员用户会看到 **📊 Dashboard** 入口（普通用户无此选项）
5. 点击 **Dashboard** 进入管理后台

### 管理后台布局

后台左侧菜单栏：

| 菜单 | 功能 |
|------|------|
| 📊 **Dashboard** | 统计数据概览（产品/订单/用户/收入/待审评论） |
| 🏛️ **Tours** | 产品列表 → 可编辑/管理团期/查看 |
| 📋 **Orders** | 订单列表 → 可修改状态 |
| ⭐ **Reviews** | 评论审核 → 审批/拒绝 |

### 4.1 查看产品列表

1. 进入 **Dashboard** → 点击左侧 **Tours**
2. 表格显示：名称、状态、价格、天数、难度
3. 每行操作按钮：
   - **Edit** → 编辑产品基础信息
   - **Dates** → 管理团期
   - 👁️ → 前台查看

### 4.2 编辑产品（全部字段可修改）

**典型场景**：调整价格、修改名称描述、更新亮点、替换图片

1. 在 **Tours** 列表 → 点击该产品的 **Edit** 按钮
2. 编辑页面按区块分为 4 个区域，**均可编辑**：

**① Basic Information**（基本信息）

| 字段 | 可编辑 | 说明 |
|------|--------|------|
| Slug | ❌ 只读 | 产品标识，创建后不可改 |
| Status | ✅ | Draft（草稿）/ Published（已发布） |
| Type | ✅ | Group Tour / Private Tour |
| Difficulty | ✅ | Easy / Moderate / Challenging |
| Duration (Days) | ✅ | 行程天数 |
| Duration (Nights) | ✅ | 住宿晚数 |
| Start Price ($) | ✅ | 起价 |
| Currency | ✅ | USD / CNY / EUR |
| Max Pax / Min Pax | ✅ | 成团人数限制 |

**② Name & Description**（名称与描述 — 支持多语言编辑）

编辑区域上方有 **English / 中文 / Español** 三个标签页，点击可切换当前编辑的语言版本。

| 字段 | 说明 |
|------|------|
| Name * | 产品名称（当前标签页的语言版本） |
| Subtitle | 副标题（当前标签页的语言版本） |
| Description | 产品详细描述，多行文本（当前标签页的语言版本） |

> 切换标签页会显示对应语言的已有内容。所有语言的修改会一同保存。有内容的语言版本在标签上会显示绿色圆点指示。

**③ Features**（亮点 / 包含 / 不包含 — 支持多语言编辑）

同样，Features 区域上方也有 **English / 中文 / Español** 三个标签页。

- 每项可 **增**（点 Add）+ **删**（点 🗑️）+ **改**（直接编辑文本）
- 切换标签页可编辑不同语言版本
- 有内容的语言版本在标签上会显示绿色圆点指示

**④ Photos & Videos**（图片和视频管理）— 详见 4.3 节

3. 所有修改完成后点击右上角 **Save Changes**
4. 页面显示绿色成功提示 → 修改已生效 ✅

### 4.3 图片和视频管理

**编辑页面底部**的 Photos & Videos 区域，支持：

| 功能 | 操作 |
|------|------|
| 📤 **上传** | 点击 **Upload Media** 按钮 → 选择文件 |
| 🖼️ **图片预览** | 自动生成缩略图，支持 JPG/PNG/WebP/GIF/SVG |
| 🎬 **视频预览** | 显示播放器界面 + 🎬 标识，支持 MP4/WebM/MOV |
| 🗑️ **删除** | 鼠标悬停到媒体上 → 红色删除按钮 → 确认删除 |
| ⚙️ **文件大小** | 单文件 ≤ 60MB |

**支持的文件格式**：

```
图片: .jpg .jpeg .png .webp .gif .svg
视频: .mp4 .webm .mov（≤60秒，≤60MB）
```

> 上传后的图片和视频立即显示在产品详情页。视频在页面中展示为可点击播放的媒体元素。

### 4.4 创建新产品

1. **Tours** 列表 → 点击 **Add Tour**
2. 填写表单：
   - **Basic Information**: Slug、状态、类型、价格等
   - **Translations**: 至少填写一个语言版本（名称必填）
   - **Features**: 亮点、包含、不包含
   - **Images**: 输入 URL 或上传图片
   - **Tour Dates**: 添加团期（日期、价格、名额）
3. 点击 **Create Tour** → 创建成功自动返回列表

### 4.5 高级操作：通过 API 完整更新产品

运维人员可通过直接调用 API 批量更新产品信息（含翻译、图片、团期）：

```bash
# 获取管理员 token
TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@echotours.com","password":"Admin123!"}' \
  | python3 -c "import json,sys;print(json.load(sys.stdin)['access_token'])")

# 获取产品完整详情
curl -s http://localhost:8000/api/v1/admin/tours/{TOUR_ID}?locale=en \
  -H "Authorization: Bearer $TOKEN"

# 完整替换更新（PUT 方式，会替换所有翻译/图片/团期）
curl -X PUT http://localhost:8000/api/v1/admin/tours/{TOUR_ID} \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "start_price": 109.0,
    "difficulty": "moderate",
    "translations": [
      {"locale": "zh", "name": "故宫深度游"},
      {"locale": "en", "name": "Forbidden City Tour"}
    ],
    "dates": [
      {"start_date": "2026-07-01", "end_date": "2026-07-01",
       "price_per_pax": 109.0, "availability": 20}
    ]
  }'
```

## 5. 团期管理（运营核心功能）

团期管理是运营人员最常用的功能。所有修改**即改即生效**（Redis 缓存 60 秒内自动刷新）。

### 5.1 进入团期管理

**路径**: 管理后台 → **Tours** 列表 → 点击产品的 **Dates** 按钮

### 5.2 修改价格或库存（高频操作）

**场景**：旺季涨价、淡季促销、名额不足时调整

1. 在团期表格中找到目标日期行
2. 点击该行右侧的 **Edit** 按钮
3. **Price** 列变为可编辑输入框 → 输入新价格
4. **Available** 列变为可编辑输入框 → 输入新名额
5. 点击 **✓** 保存 → 页面刷新，显示绿色 "Date updated!" 提示 ✅

> 当库存 ≤ 3 时，数字会高亮为琥珀色，提醒运营人员注意。

### 5.3 新增团期

**场景**：新产品上线、增加旺季班次

1. 在团期页面顶部蓝色 **Add New Date** 区域：
   - **Date**：选择出发日期
   - **Price ($)**：单人价格
   - **Spots**：可用名额
2. 点击 **Add** → 新团期立即出现在表格中 ✅
3. 前台用户即刻可以看到并预订

### 5.4 下架团期（售罄/取消）

**场景**：团期售罄、行程取消

**方法一（推荐 — 保留记录）**：
1. 点击该团期 **Edit** → 将 **Available** 改为 `0`
2. 点 **✓** 保存 → 前台显示 "Sold Out" 但后台保留记录

**方法二（彻底删除）**：
1. 点击该团期右侧的 **🗑️** 图标
2. 弹出确认对话框 → 点 **确定**
3. 该团期从数据库永久删除，不可恢复

### 5.5 批量调整

对于大批量团期调整，运维人员可以通过 API 脚本批量处理：

```bash
# 示例：将某产品所有团期涨价 $10
docker compose exec backend python3 -c "
import httpx, json

r = httpx.post('http://localhost:8000/api/v1/auth/login',
    json={'email':'admin@echotours.com','password':'Admin123!'})
token = r.json()['access_token']
headers = {'Authorization': f'Bearer {token}'}
TOUR_ID = '替换为实际产品ID'

# 获取所有团期
r = httpx.get(f'http://localhost:8000/api/v1/admin/tours/{TOUR_ID}/dates',
    headers=headers)
dates = r.json()['dates']

# 统一涨价 $10
for d in dates:
    if d['status'] == 'available':
        new_price = round(d['price_per_pax'] + 10, 2)
        r = httpx.patch(f'http://localhost:8000/api/v1/admin/tours/{TOUR_ID}/dates/{d[\"id\"]}',
            headers=headers,
            json={'price_per_pax': new_price})
        print(f\"  {d['start_date']}: \${d['price_per_pax']} → \${new_price} {'✅' if r.status_code == 200 else '❌'}\")
"
```

## 6. 订单管理

### 查看订单

1. 管理后台 → 左侧 **Orders**
2. 表格显示：订单号、联系人、金额、状态、支付状态
3. 可按状态筛选

### 修改订单状态

1. 订单行点击操作按钮
2. 可设置状态：`Pending` → `Confirmed` → `In Progress` → `Completed`
3. 支付状态：`unpaid` → `paid` → `refunded`

> 订单状态变更后即刻生效。退款等需要财务确认的操作建议在外部系统完成后再更新系统状态。

## 7. 评论审核

1. 管理后台 → 左侧 **Reviews**
2. 默认显示待审核（**Pending**）评论

### Q: 如何收藏产品并在何时可以看到收藏按钮？

### Q: 如何查看开发环境中发出的邮件？

开发环境使用 MailHog 捕获所有外发邮件（无需 SendGrid 密钥）：

```bash
# 启动 MailHog（首次需加 profile 参数）
docker compose --profile mail up -d

# 访问 MailHog Web 界面
open http://localhost:8025
```

MailHog Web 界面显示所有被拦截的邮件，支持查看 HTML 内容和 JSON 原始数据。
适合调试：欢迎邮件、预订确认邮件、评价通知邮件等。

### Q: 访问 `http://localhost` 返回 "502 Bad Gateway"？

**原因**：Nginx 代理无法连接到 frontend 或 backend 容器。通常是 Docker 网络问题，常见于混合使用了 `docker-compose.yml` 和 `docker-compose.prod.yml` 两套配置文件。

**解决方法**：

```bash
# 1. 检查所有服务是否在运行
docker compose ps

# 2. 如果某个服务未运行，重启它
docker compose up -d <service_name>

# 3. 如果 frontend/backend 在运行但 nginx 报 502，检查网络
#    nginx 和 frontend/backend 必须在同一 Docker 网络内
docker inspect echo-website-nginx-1 --format '{{json .NetworkSettings.Networks}}'
docker inspect echo-website-frontend-1 --format '{{json .NetworkSettings.Networks}}'

# 4. 如果网络不一致，重启 nginx 修复
docker compose up -d nginx

# 5. 如果问题依旧，完整重启
docker compose down && docker compose up -d
```

> ⚠️ **不要**混用 `docker-compose.yml` 和 `docker-compose.prod.yml`。日常开发只使用 `docker compose`（默认 dev 配置）。

### Q: 修改团期价格后，前台没有立即生效？

团期数据有 60 秒 Redis 缓存。修改后最多等 1 分钟即可刷新。如果需要立即清理缓存：

```bash
docker compose exec redis redis-cli KEYS 'cache:TourService:get_tour_dates:*' | xargs -r docker compose exec redis redis-cli DEL
```

### Q: 如何添加新的管理后台管理员？

通过 API 或数据库直接修改用户的 `is_admin` 字段：

```bash
docker compose exec backend python3 -c "
import asyncio, uuid
from app.database import async_session
from sqlalchemy import select, update
from app.models.user import User

async def make_admin(email):
    async with async_session() as db:
        await db.execute(
            update(User).where(User.email == email).values(is_admin=True)
        )
        await db.commit()
    print(f'{email} 已设置为管理员')

asyncio.run(make_admin('zhangsan@example.com'))
"
```

### Q: 前端页面显示空白/报错？

检查前端容器日志：

```bash
docker compose logs frontend | tail -30
```

常见原因：
- Next.js 开发模式下首次编译较慢（等待 30 秒左右）
- API 代理未正确配置（检查 `next.config.ts` 中的 rewrites）
- 后端不可用（检查 `docker compose ps`）
