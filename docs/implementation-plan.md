## 1. 项目实施路径

### 1.1 阶段1：MVP（0-6个月）

**目标**：验证核心业务模型，获取首批付费用户

#### 里程碑
| 里程碑 | 时间 | 交付物 |
|--------|------|--------|
| M1.1 需求确认 | 第1-2周 | PRD文档 + UI设计稿 |
| M1.2 技术基建 | 第3-4周 | 项目框架 + CI/CD + 云环境 |
| M1.3 核心功能 | 第5-10周 | 产品展示 + 搜索 + 预订 + 支付 |
| M1.4 多语言 | 第11-12周 | 中/英/西三语上线 |
| M1.5 Alpha测试 | 第13-14周 | 内部测试 + Bug修复 |
| M1.6 Beta上线 | 第15-16周 | 小范围内测 + 邀请首批种子用户 |
| M1.7 正式发布 | 第17-24周 | 公开上线 + 市场推广 |

#### MVP 功能范围
- 产品浏览与搜索
- 旅游产品详情页（图文+行程表）
- 在线预订与支付（Stripe）
- 用户注册/登录
- 订单管理
- 多语言（中文/英文/西班牙语）
- CMS内容管理
- 移动端响应式Web

## 5. AWS vs GCP 成本对比与建议

### 5.1 成本对比

```
年度云基础设施费用对比（USD/年）

                    AWS                 GCP
阶段1 MVP     $4,740 - $7,680    $4,080 - $6,540
阶段2 增长    $13,080 - $20,040  $12,360 - $19,620  
阶段3 规模化   $36,360 - $61,200  $34,680 - $61,680

GCP 月成本约比 AWS 低 10-15%，尤其在 MVP 阶段优势明显。
```

### 5.2 详细优势对比

| 维度 | AWS 优势 | GCP 优势 |
|------|---------|---------|
| **计算成本** | — | Cloud Run 冷启动略快，闲置不计费 |
| **数据库** | RDS 成熟度更高，Multi-AZ 稳定 | Cloud SQL 性价比略高 |
| **容器** | ECS Fargate 简洁 | GKE Autopilot 更灵活(但更贵) |
| **AI/ML** | SageMaker 复杂 | **Vertex AI 集成度极高（优势）** |
| **翻译服务** | Translate 稳定 | **Cloud Translation API 性价比更高** |
| **CDN** | CloudFront 全球节点最多 | **Cloud CDN + Premium Tier 出站费低** |
| **免费额度** | 12个月有限免费 | **$300新用户赠金 + 部分服务永久免费层** |
| **学习曲线** | 服务繁多，配置复杂 | 更简洁统一 |

### 5.3 推荐方案

#### 推荐：**AWS 主用 + GCP AI 补充**

**理由：**
1. AWS 的生态更成熟，旅游行业案例更多（Expedia 使用 AWS）
2. GCP 的 Vertex AI 和 Translation API 对多语言+AI非常重要
3. 多云策略避免供应商锁定
4. 中国区部署需求（如需）: 阿里云或 AWS 中国区

**具体部署建议：**

```
初期推荐架构（混合云）:
┌──────────────────────────────────────────────────┐
│  AWS (主要业务运行)                                │
│  ├─ Web + API → ECS Fargate                      │
│  ├─ 数据库 → RDS PostgreSQL                      │
│  ├─ 缓存 → ElastiCache Redis                     │
│  ├─ 搜索 → OpenSearch Service                    │
│  ├─ CDN → CloudFront                             │
│  ├─ 文件 → S3                                    │
│  ├─ 邮件 → SES                                   │
│  └─ 监控 → CloudWatch + Grafana                  │
└──────────────────────────────────────────────────┘
    ↓ 通过私有连接或API调用
┌──────────────────────────────────────────────────┐
│  GCP (AI + 翻译)                                  │
│  ├─ AI推荐 → Vertex AI                           │
│  ├─ 翻译服务 → Cloud Translation API             │
│  ├─ AI行程规划 → Vertex AI (Gemini)              │
│  └─ 多语言NLP → Natural Language API             │
└──────────────────────────────────────────────────┘
```

## 7. 商业模式与盈利预测

### 7.1 收入模型

| 收入来源 | 模式 | 占比目标 |
|---------|------|---------|
| **产品佣金** | 从供应商收取15%-25%佣金 | 60% |
| **定制服务费** | 行程定制咨询费 $50-200/次 | 15% |
| **保险/附加服务** | 旅行保险、VIP服务等 | 10% |
| **广告/推广位** | 供应商优先展示付费 | 8% |
| **会员订阅** | 高端会员年费 $99-299/年 | 5% |

## 10. 附录：关键建议摘要

### 技术落地建议

1. **不要一上来就做App** — 先用响应式Web验证需求，P2阶段再做原生App
2. **不要直接上微服务** — 模块化单体起步，团队<8-10人不要考虑微服务
3. **不要自建支付系统** — Stripe/Adyen成熟稳定，自建成本高且合规风险大
4. **不要忽视多语言性能** — Next.js + next-intl + 数据库翻译表，注意SSR性能优化
5. **一定要做好SEO** — 旅游网站60%+流量来自搜索引擎，多语言SEO是核心

### 运营落地建议

1. **先做好一个目的地** — 初期聚焦1-2个目的地打造爆款，验证模式后再扩展
2. **内容为王** — 高质量攻略/游记/视频 = 免费SEO流量
3. **AI辅助内容** — 用AI生成多语言内容草稿 + 人工精校
4. **供应商关系** — 与其平台抽佣，不如帮供应商做好数字化转化
5. **客服体验 = 核心竞争** — 定制旅游的决胜点在服务体验

### 云成本建议

1. **MVP阶段：AWS = $400-650/月** | 用预留实例可再省30%
2. **增长期：AWS = $1,000-1,700/月** | 配合Auto Scaling + 缓存优化
3. **规模化：AWS = $3,000-5,100/月** | 考虑预留实例+Spot实例
4. **省钱技巧**：
   - 使用T系实例（可突增）+ CPU Credits
   - 非生产环境使用定时开关机
   - 闲置资源自动回收
   - S3生命周期管理（30天→IA，90天→Glacier）
