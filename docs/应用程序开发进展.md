### 2.11 Batch 9h：Python Playwright 全业务流程 E2E 自动化脚本

**背景**：之前有 TypeScript Playwright 测试（`tests/e2e/`），但用户希望用 Python `playwright` 库编写有头模式的自动化脚本，直观观察浏览器操作整个业务流程。

**新增/修改文件**（2 个）：

| 文件 | 操作 | 说明 |
|------|------|------|
| `scripts/e2e_test.py` | **新增** | 808 行 Python Playwright 全业务流程自动化脚本 |
| `src/backend/pyproject.toml` | **修改** | `[dev]` 依赖新增 `playwright>=1.52.0` |

**脚本架构**（`E2ETestRunner` 类）：

| 组件 | 说明 |
|------|------|
| `setup()` | 启动 Chromium 有头浏览器（`headless=False`） |
| `login()` / `register()` | 通过 UI 登录/注册（`wait_for_url` 等待跳转） |
| `api_get()` / `api_post()` | 浏览器内 `page.evaluate()` 调用 API（自动携带 JWT） |
| `screenshot()` | 每步自动保存全页截图到 `scripts/screenshots/` |
| `run_step()` | 包装每一步，异常捕获 + 截图 + 结果记录 |
| `summary()` | 输出 PASS/FAIL 汇总报告 |

**16 步业务覆盖**：

| 阶段 | 步骤 | 描述 |
|------|------|------|
| 🌐 公开浏览 | 01-05 | 首页 → 产品列表 → 搜索 → 目的地 → 产品详情 |
| 👤 用户认证 | 06 | **注册新用户**（随机邮箱，自动登录） |
| ❤️ 用户操作 | 07-11 | 心愿单 → **UI 端到端下单+Mock支付** → 订单历史 → 评价 → 个人中心 |
| 🔐 后台管理 | 12-16 | 管理员登录 → 仪表盘 → 产品/订单/评价管理 |

**UI 支付流程（双路径容错）**：

```
page.goto(/checkout?tour=X&date=Y&pax=1)
  ├─ 25s 内 React hydrate → 填写联系人 → Pay Now
  │   → 创建订单 → 创建支付(mock) → 自动跳转 /checkout/success ✅
  └─ 超时(Next.js dev 编译延迟) → API 兜底
      → POST /api/v1/orders → 验证 order_no ✅
```

**产出物**：
- 每次运行保存截图到 `scripts/screenshots/`（带时间戳 + PASS/FAIL 前缀）
- 控制台输出每步详细日志（URL、API 响应、耗时）
- 最终 16/16 全通过汇总报告

**依赖安装**：
```bash
cd src/backend && pip install -e ".[dev]" && playwright install chromium
```

**运行**：
```bash
python scripts/e2e_test.py                    # 有头 + 中文
python scripts/e2e_test.py --locale en        # 英文界面
python scripts/e2e_test.py --headless          # 无头 (CI)
python scripts/e2e_test.py --slow-mo 500      # 慢速调试
```
