# Echo Tours 全业务流程测试用例

## 一、概述

本文档覆盖 Echo Tours 旅游平台的完整业务流程测试用例，包括：
- **B2C 客户旅程**：注册 → 登录 → 浏览产品 → 收藏 → 下单 → 支付 → 评价
- **B2B 管理流程**：管理员登录 → 统计看板 → 产品 CRUD → 订单管理 → 评论审核
- **公共浏览**：目的地浏览 → 景点查看 → 全文搜索
- **用户中心**：个人资料管理 → 订单查询 → 收藏管理
- **E2E 全流程**：20 个 Playwright spec 文件覆盖前端 SSR 渲染/多语言/结账/搜索/评价/管理/回归/后置清理等

**测试总览**：
| 类别 | 数量 |
|------|------|
| 🧪 后端 API 测试（pytest） | **467 项全部通过** |
| 🌐 E2E 测试（Playwright） | **20 个 spec 文件，回归/逆向/多语言/后置清理全覆盖** |
| 🐍 Python 自动化（Playwright Python） | **16 步全业务流程通过** |

## 二、测试范围与策略

### 测试层级
| 层级 | 范围 | 工具 |
|------|------|------|
| 单元测试 | CRUD/Service 层 | pytest + pytest-asyncio |
| API 集成测试 | 完整 HTTP 接口 | httpx.AsyncClient（34 个测试文件） |
| E2E 测试 | 前端渲染+交互+后置清理 | Playwright (20 spec 文件) |
| Python 自动化 | 全业务流程有头模式 | Playwright Python |

### 测试策略
- 所有 API 集成测试通过 httpx 直连运行中的 Docker 服务
- CRUD 测试直接使用数据库 session
- 每个测试独立事务，测试后自动回滚
- E2E 使用 Playwright Chromium（1280×800，30s 超时），失败自动截图
- 测试数据库通过 `auto_cleanup` fixture 自动隔离，避免测试间干扰
